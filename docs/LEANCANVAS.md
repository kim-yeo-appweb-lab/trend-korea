# 트렌드 코리아 린 캔버스 (Lean Canvas)

> 작성일: 2026-03-09 | 기반 문서: [PRD](./PRD.md)

## ① 문제 (Problem)

> PRD 1.2 해결하는 문제

1. **뉴스 파편화** — 뉴스가 여러 매체에 흩어져 있어 하나의 사건을 시간순으로 파악하기 어렵다
2. **추적 불가** — 한 번 보도된 사건의 후속 경과를 추적하려면 직접 검색해야 한다
3. **토론 공간 부재** — 사건에 대한 맥락 있는 토론 공간이 부족하다

## ② 솔루션 (Solution)

> PRD 3장 메인 시나리오, 7장 기능 요구사항, 9장 데이터 수집 파이프라인

| 솔루션 | 문제 매핑 | 핵심 기능 |
|--------|----------|----------|
| 타임라인 | 뉴스 파편화 | 일자별 사건 아카이브, 필터/정렬, 무한 스크롤, 상세 모달 |
| 이슈 추적 | 추적 불가 | 이슈 상태 관리, 트리거 타임라인, 추적 등록/알림 |
| 커뮤니티 | 토론 공간 부재 | 사건 맥락 기반 토론, 게시글/댓글/추천 |
| AI 뉴스 파이프라인 | 전체 | 자동 키워드 수집 → 크롤링 → 분류 → 요약 (15분 주기) |

## ③ 가치 제안 (Value Proposition)

> PRD 1.1 한 줄 정의, 1.3 핵심 가치

- **고유 가치**: 흩어진 뉴스를 시간순으로 정리하고, 이슈의 후속 경과를 자동 추적하며, 맥락 있는 토론 공간을 제공
- **상위 개념**: 대한민국 사건/이슈의 시간순 기록 + 지속 추적 정보 아카이브

## ④ 고객 세그먼트 (Customer Segments)

> PRD 2.1~2.3 페르소나

| 세그먼트 | 페르소나 | 핵심 동기 |
|---------|---------|----------|
| 뉴스 소비자 | 김민수 (29세, 직장인) | "오늘 무슨 일이 있었는지 5분 안에 파악하고 싶다" |
| 이슈 추적자 | 이수진 (34세, 프리랜서 기자) | "관심 이슈의 상태 변화를 놓치지 않고 싶다" |
| 운영자 | 1인 개발자 겸 운영 | "자동화된 파이프라인으로 운영 부담을 최소화하고 싶다" |

## ⑤ 경쟁우위 (Unfair Advantage)

1. **AI 자동화 파이프라인** — 15분 주기로 키워드 수집 → 뉴스 크롤링 → 분류/중복 제거 → OpenAI 요약 → 피드 저장
2. **사건-이슈-트리거 연결 구조** — 단발성 사건이 아닌 이슈의 맥락을 트리거로 연결하여 추적
3. **통합 플랫폼** — 사건 아카이브 + 이슈 추적 + 커뮤니티를 하나의 서비스로 통합

## ⑥ 채널 (Channels)

- **웹 서비스** — 모바일 퍼스트 반응형 (Next.js 16 SSR)
- **SEO** — SSR + Open Graph 메타태그로 검색 엔진 노출 극대화
- **트렌드 키워드 유입** — 실시간 검색순위/트렌드 키워드 기반 자연 유입
- **커뮤니티 입소문** — 사건 맥락 기반 토론 활성화를 통한 유기적 확산
